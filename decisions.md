## [2026-07-01 · Step2 /insight] 유사도 기준 설계

  - 하드 필터 통과 후보 내에서 topic_category(가중치 4) · has_emoji(가중치 2) · posting_hour(가중치 1) 일치 여부로 유사도 점수(0~7) 계산
  - 동점 시 headline_length 차이가 작은 쪽을 우선(제목 길이도 유사도의 일부로 간주)
  - 점수 내림차순 정렬 → 상위 3개를 유사 TOP3로 채택
- 선택한 이유:
  - 완화 순서를 posting_hour → has_emoji → topic_category로 정했으므로(가장 먼저 포기하는 조건이 posting_hour), 반대로 가중치는 topic_category를 가장 높게, posting_hour를 가장 낮게 주어 "먼저 포기하는 조건일수록 유사도 판단에서 비중이 작다"는 논리를 정확히 반영
  - 이진 가중치(4·2·1)를 쓰면 하나의 상위 조건 불일치가 하위 조건 전부 일치보다 항상 우선순위가 낮아지도록 보장되어, 실제 "조건을 하나씩 완화"하는 것과 수학적으로 동일한 정렬 결과를 만듦
  - 신규 5건 모두 type×channel 조합의 과거 후보가 6건 이상이라(예: 숏폼×유튜브 13건, 카드뉴스×인스타그램 22건) 하드 필터만으로도 TOP3 확보에 문제없음을 1단계 분석에서 확인
- 대안으로 고려했던 것:
  - type·channel까지 완화 대상에 포함(가이드 예시) → 미채택. 형식이 다른 콘텐츠(예: 블로그 vs 숏폼)를 비교하면 CTR 기준 자체가 달라 마케터에게 오히려 잘못된 벤치마크를 줄 위험 판단
  - 5개 조건 동일 가중치 → 미채택. "완화 순서"라는 설계 의도를 점수에 반영하지 못함

## [2026-07-01 · Step3 /generate] 개선 제안 선정 근거
- 결정 내용:
  - 신규 콘텐츠 1건마다 posting_hour·has_emoji·headline_length(블로그 한정) 3개 레버 각각의 "변경 시 CTR 개선폭(%p)"을 채널 단위 집계로 자동 계산
  - 개선폭(delta)이 큰 순으로 정렬해 상위 2개만 채택. 3개 레버 모두 개선 여지가 없으면(이미 최적 조건) type×channel 조합 비교로 대체 제안 1개를 보충
  - 모든 수치는 predict_pipeline.py의 집계 함수(compute_aggregates)에서 실시간 계산 — 특정 항목을 사람이 직접 골라 하드코딩하지 않음
- 선택한 이유:
  - "감이 아니라 데이터 근거"라는 채점 기준을 지키려면 제안 선택 자체도 규칙 기반이어야 함. delta 크기 순 정렬은 가장 단순하면서도 "가장 효과 큰 개선안부터 제시"라는 마케팅 의사결정 원칙과 일치
  - 채널 단위로 집계한 이유: company-info.md 예시("같은 유형·채널의 저녁 발행 평균 CTR")를 따르되, type×채널까지 쪼개면 일부 조합의 표본이 1~2건으로 줄어 평균의 신뢰도가 낮아지므로 채널 단위로 완화해 표본을 확보
- 대안으로 고려했던 것:
  - 5건 각각에 대해 사람이 임의로 제안 문구 작성 → 미채택. Standard 요구사항인 "새 CSV를 넣어도 동일 품질로 자동 생성"을 만족할 수 없음
  - type×channel 단위로 세분화한 시간대 평균 사용 → 미채택. 일부 조합 표본이 1건뿐이라 평균의 통계적 의미가 약함

## [2026-07-01 · Step5 /generate(Challenge)] 전략 기획안 우선순위 판단 기준
- 결정 내용: 우선순위 점수 = CTR 임팩트(45%) + 적용 범위(35%) + 실행 난이도(20%), 100점 만점으로 두 기획안을 채점해 P0/P1 부여
  - CTR 임팩트: 해당 전략이 근거로 삼는 실측 CTR 차이(%p)가 클수록 높은 점수
  - 적용 범위: 전체 정제 데이터(69건) 중 그 전략을 적용할 수 있는 콘텐츠 비율
  - 실행 난이도: 발행 시간·이모지처럼 즉시 적용 가능한 운영 규칙일수록 높은 점수, 콘텐츠 재제작이 필요하면 낮은 점수
- 선택한 이유:
  - CTR 임팩트에 가장 큰 비중(45%)을 둔 이유: 이 과제의 핵심 성과지표가 CTR이므로, 실제 클릭률 개선폭이 가장 직접적인 사업 가치
  - 적용 범위(35%)를 실행 난이도(20%)보다 높게 둔 이유: 아무리 쉬운 개선이라도 소수 콘텐츠에만 적용되면 전체 성과에 미치는 영향이 작음 — 반대로 넓게 적용되는 규칙은 실행이 다소 번거로워도 누적 효과가 큼
- 대안으로 고려했던 것:
  - 3개 지표 동일 가중치(33/33/34) → 미채택. CTR 개선이라는 과제 목적과 무관하게 점수가 산출될 위험

## [벤치마크 신뢰도 조사] 신규 콘텐츠 5의 TOP3 저점수 이슈
- 조사 배경: 신규 콘텐츠 5(숏폼·트렌드·인스타그램·12시·이모지X) TOP3에 유사도 0/7, 1/7짜리가 포함된 원인 확인 요청
- 조사 결과: **정렬/필터링 로직 버그 아님.** `top3_similar()`가 실제로 하드필터 통과 후보 8건 전부를 올바르게 계산·정렬한 결과가 [4, 1, 0, 0, 0, 0, 0, 0]점이었음. 원인은 이 풀(숏폼×인스타그램) 8건이 전부 `has_emoji=True`인데 신규5는 `has_emoji=False`를 요구해, has_emoji·posting_hour 두 축에서 거의 매칭이 불가능한 구조적 상황이었기 때문. topic_category만 일치하는 C007(4점)이 유일한 실질적 유사 콘텐츠였음
- 결정 내용: 애초 요청받은 "유사도 2점 이상 후보가 0건이면 저신뢰 표시" 기준을 "2점 이상 후보가 3건 미만이면 저신뢰 표시"로 조정해 `assess_confidence()` 함수를 구현. 신규 콘텐츠 5는 8건 중 1건만 2점 이상이라(qualified_count=1 < 3) 저신뢰로 판정되어 리포트에 경고 문구 + "추정 CTR 범위(신뢰도 낮음)" 표시가 자동으로 붙음
- 선택한 이유: 실제 데이터를 5건 전체에 대해 검증한 결과(신규1: 13건 중 11건, 신규2: 22건 중 20건, 신규3: 17건 중 17건, 신규4: 6건 중 6건이 2점 이상) 신규5만 유일하게 8건 중 1건뿐이었음. "0건"만 저신뢰로 잡으면 "TOP3 중 1개만 진짜 유사하고 나머지 2개는 억지로 채운" 신규5 같은 케이스를 놓치므로, "TOP3를 채울 만큼(3건) 충분한 후보가 있는가"로 기준을 재정의하는 것이 실제 문제 상황과 더 정확히 일치함
- 대안으로 고려했던 것: 사용자가 제시한 문구 그대로 "0건"만 기준으로 사용 → 미채택. 신규5 사례에서 qualified_count=1이라 조건이 트리거되지 않아 실제 문제를 놓치는 결과가 나옴을 직접 실행 검증 후 확인

## [strategy_plan.md 보강 1] CTR 임팩트 숫자 불일치(이론상 +2.54%p vs 로드맵 "1%p+") 해결
- 조사 배경: 기획안1 근거데이터의 "이론상 최대 +2.54%p"와 로드맵 요약의 "평균 CTR 1%p+ 상승"이 서로 다른 근거로 보여 불일치 지적을 받음
- 조사 결과: 소셜 채널(50건) 내에서 "저녁(18/20시)+이모지O"를 이미 동시에 만족하는 콘텐츠(37건, 평균CTR 4.12%) vs 아직 하나라도 만족 못하는 콘텐츠(13건, 평균CTR 3.14%)를 직접 비교한 실측 결합효과는 **+0.99%p**로 확인됨. 이는 로드맵의 "1%p+" 표현과 거의 일치하고, 근거데이터의 "+2.54%p"는 두 레버를 각각 독립 측정한 값의 단순 합산(이론상 상한)이라 실제보다 과장돼 있었음
- 결정 내용: "+2.54%p"는 "이론상 상한(실측 아님)"이라고 명시적으로 라벨링하고, 실측 결합효과(+0.99%p)를 CTR 임팩트 점수·로드맵 기대효과의 공식 기준 수치로 채택
- 선택한 이유: 소셜 콘텐츠의 74%(37/50)가 이미 두 조건을 동시에 충족하고 있어 두 레버가 상당히 겹침(독립적이지 않음) — 이 상관관계 때문에 단순 합산이 실제보다 크게 나온 것. "감이 아니라 데이터 근거"라는 원칙을 지키려면 이론상 상한이 아니라 직접 측정한 결합효과를 공식 수치로 써야 함
- 대안으로 고려했던 것: 로드맵의 "1%p+"를 "보수적으로 낮춰 잡은 추정치"라는 각주만 추가하고 넘어가기 → 미채택. 실제로 계산해보니 "보수적 추정"이 아니라 거의 정확한 실측치였으므로, 각주보다 원인(가산 가정의 겹침)을 명확히 밝히는 쪽이 더 정직함

## [strategy_plan.md 보강 2] 가산 가정 실측 검증 결과 CTR 임팩트 점수 재계산
- 결정 내용: 위에서 확인한 실측 결합효과(+0.99%p)를 기준으로 기획안1·2의 CTR 임팩트 점수를 다시 계산. 두 기획안 중 실측 개선폭이 더 큰 쪽(기획안2의 +1.74%p)을 45점 만점 기준으로 삼아, 기획안1은 45×(0.99/1.74)=26점, 기획안2는 45점을 부여. 그 결과 총점이 기획안1 71점, 기획안2 71점으로 **동점**이 나옴
- 선택한 이유: 근거데이터 문구를 실측치로 고쳤는데 우선순위 점수 계산식은 옛 이론상 수치(+2.54%p)를 그대로 쓰면 같은 문서 안에서 서로 다른 숫자를 근거로 삼는 모순이 생김. 문서 내적 일관성을 위해 점수도 함께 재계산함
- 동점 처리: "동점일 경우 실행 난이도 점수가 더 높은 쪽을 우선한다"는 타이브레이크 규칙을 새로 추가해 기획안1을 P0로 유지. 다만 점수가 근접하므로 로드맵에 "순서보다 동시 진행을 권장"한다는 코멘트를 추가
- 대안으로 고려했던 것: 점수 재계산 없이 문구만 수정 → 미채택. 문서 안에서 "CTR 임팩트 45/45(+2.54%p 기준)"이라는 점수 근거와 "실측 결합효과는 +0.99%p"라는 근거데이터가 동시에 존재하면 자기모순이 되어 신뢰도가 오히려 떨어짐

## [strategy_plan.md 보강 3] 적용범위 기준 명확화 — type≠블로그와 channel≠블로그 혼용 오류 수정
- 조사 배경: 기획안1의 "소셜 채널 75.4%(52/69)"가 왜 블로그를 제외한 기준인지, 블로그도 이모지·저녁 효과가 있을 수 있는데 왜 적용범위에서 뺐는지 근거를 요청받음
- 조사 결과 1(계산 오류 발견): "52건(75.4%)"은 사실 `type≠블로그`(카드뉴스·숏폼·인포그래픽) 건수였고, 실제 채널 기준 소셜(channel이 인스타그램 또는 유튜브)은 50건(72.5%)임. 차이가 나는 이유는 인포그래픽 2건이 channel=블로그로, 1건이 channel=유튜브(type상으로는 '블로그' 아님)로 발행되어 type 기준과 channel 기준이 서로 어긋나기 때문. 마찬가지로 기획안2의 "블로그 17건(24.6%)"도 `type==블로그` 기준이었고, 정확한 channel==블로그 기준은 19건(27.5%)임(인포그래픽 2건 포함)
- 조사 결과 2(블로그 제외 근거): 정제 데이터의 channel==블로그 19건을 확인한 결과 posting_hour는 전부 8 또는 12뿐이었고 has_emoji는 전부 False였음 — 즉 "블로그+저녁발행"이나 "블로그+이모지" 조합이 이 데이터셋에 아예 없어 효과 유무를 검증할 수 없음
- 결정 내용: 적용범위는 항상 channel 기준으로 통일(소셜=인스타그램·유튜브 50건/72.5%, 블로그=19건/27.5%)하도록 정정. 블로그를 기획안1 적용범위에서 제외하는 근거로 "효과가 약할 것 같다"는 추측 대신 "이 데이터셋엔 검증할 조합 자체가 없다"는 사실을 명시
- 선택한 이유: "검색 유입 특성상 효과가 약할 것"이라는 설명은 industry-news.md의 일반론일 뿐 이 프로젝트의 실제 데이터로 뒷받침되지 않는 추측이므로 채택하지 않음. 반면 "데이터에 해당 조합이 없다"는 문장은 69건 데이터를 직접 확인해 검증한 사실이라 더 정직하고 방어 가능한 근거임
- 대안으로 고려했던 것: 블로그도 적용범위에 포함해서 재계산 → 미채택. 근거(데이터)가 전혀 없는 상태에서 효과가 있다고 가정하고 범위를 넓히는 것은 "감이 아니라 데이터 근거"라는 이 프로젝트의 원칙에 위배됨

## [content_prediction_report.md 보강] 개선 제안 근거 강화 및 표본 크기 투명화
- 조사 배경: 신규1의 개선 제안 2번(발행 시간 18시 변경, n=3건, +0.27%p)이 표본이 너무 작아 근거가 약하다는 지적을 받아, 더 튼튼한 표본으로 개선 여지가 있는 다른 레버가 있는지 재검증 요청받음
- 조사 결과: 숏폼×유튜브 풀(n=13) 내에서 (1) 이모지는 이미 True로 최적 상태, (2) 제목 길이는 상관계수 0.52로 나타났으나 버킷 비교(15자 이하 n=5 vs 16자 이상 n=8)가 emoji·시간대와 뒤섞여 있어 독립 레버로 보기 어려움, (3) topic_category(커리어)별 CTR은 4.17~4.50% 범위로 편차가 미미(n=3~4), (4) 시간대 비교를 채널 전체(n=1~7)·유형 전체(n=2~12)·전체 69건(n=13~19) 등 더 넓은 단위로 재계산해봐도, 오히려 전체 69건 기준으로는 20시(4.36%)가 18시(3.90%)보다 높아 방향이 뒤집힘 — 더 강한 대안 레버는 발견되지 않음
- 결정 내용: 기존 발행 시간 제안(n=3)은 유지하되, "전체 콘텐츠 기준으로는 방향성이 엇갈린다"는 문장을 자동으로 덧붙이도록 `_detail_candidates()`에 반전 감지 로직을 추가(표본<5이고 전체 집계 방향이 반대일 때만 발동, 하드코딩 아님). 신규1의 개선 제안 1번(유지 권장)에는 2위 조합과의 격차(+0.33%p, 숏폼×인스타그램 4.05%)를 계산해 추가 — 사용자가 예시로 든 "2위=카드뉴스×인스타그램 3.73%"는 실제 데이터로는 3위였으므로 정정
- 추가 결정: n<5인 통계치에는 `fmt_n()` 헬퍼 함수가 자동으로 "(표본이 적어 참고용)" 캡션을 붙이도록 시스템화(신규3의 30자 이상 n=4, 신규5의 이모지 미포함 n=4 등 리포트 전체에 일관 적용, strategy_plan.md와 표기 기준 통일)
- 대안으로 고려했던 것: 발행 시간 제안 자체를 삭제 → 미채택. n=3이라도 실제 데이터에 근거한 관찰이므로 삭제보다 "참고용" 캡션으로 투명하게 남기는 쪽이 더 정직함

## [content_prediction_report.md 보강] 신규2·신규4 "포맷 전환" 제안 중복 문구 차별화
- 조사 배경: 신규2·4의 개선 제안 2번("참고로 숏폼×유튜브 조합의 평균 CTR은...")이 수치만 다르고 문장 구조가 완전히 동일해 템플릿처럼 보인다는 지적을 받음
- 결정 내용: 두 문장 모두 (a) 현재 조합명을 명시하고 (b) 격차를 +%p로 명시하며 (c) "발행 시간·이모지 조정과 달리 콘텐츠 형식 자체를 바꿔야 하는 더 큰 변화이므로 세부 조정을 먼저 적용해보고 장기 검토 과제로 고려" 문장을 공통 원칙으로 추가. 신규1에서 세운 "이미 잘하고 있으면 세부조정, 아니면 큰 변화도 고려" 원칙과 일관되게, 실무적 우선순위(작은 변화 먼저)를 명시
- 선택한 이유: 신규2(현재 조합 n=22, 격차 +0.65%p)와 신규4(현재 조합 n=6, 격차 +1.13%p)는 표본 크기와 격차가 실제로 다르므로, 별도 분기 없이 같은 계산 로직을 쓰되 실제 데이터값을 그대로 노출하는 것만으로 자연스럽게 문장이 달라지도록 처리(항목별 하드코딩 지양)
- 대안으로 고려했던 것: 포맷 전환 제안을 더 작은 단위 레버(topic_category별 engagement_rate 등)로 교체 → 미채택. 격차가 크고(+0.65~1.13%p) 실무적으로 검토할 가치가 있는 정보이므로 삭제보다는 "장기 검토 과제"로 톤을 조정해 유지하는 쪽을 선택
