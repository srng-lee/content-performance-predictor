# /insight — 유사도 기준 + 패턴 설계

`/analyze` 결과를 바탕으로, 신규 5건을 진단할 **유사도 기준**과 **성과 패턴**을 설계하라.

## 1. 유사도 기준 설계
신규 콘텐츠와 과거 콘텐츠의 유사도를 정의하라. 권장 매칭 축:
- type / topic_category / channel / posting_hour / has_emoji
- 완전 일치에 가중치 부여 → 매칭 결과가 부족하면 조건을 하나씩 완화
- **조건 완화 순서**를 명시(예: posting_hour → topic_category → channel 순)하고 decisions.md에 기록

## 2. 결측·중복 처리 방침 확정
- 중복 C003: 제거 후 69건 기준으로 패턴 계산 (권장)
- 결측 C069(engagement_rate)·C070(reach): 유사 비교 시 해당 지표만 제외할지/행 제외할지 결정

## 3. 성과 패턴 도출 (개선 제안의 근거)
데이터에서 다음 같은 "차이 만드는 요소"를 수치로 정리하라:
- 발행 시간대 효과: 저녁(18/20) vs 점심(12) vs 아침(8) 평균 CTR 차이(%p)
- 이모지 효과: 같은 채널에서 has_emoji True vs False CTR 차이
- 유형×채널 궁합: 고CTR 조합 / 저CTR 조합
- 제목 길이: 소셜에서 긴 제목(30자+)의 CTR 하락 여부

## 출력
- 유사도 기준(가중치·완화 순서) 정의
- 성과 패턴 3~5개를 "요소 → CTR 차이 수치" 형태로 정리
- 다음 단계 안내: `/generate` 로 5건 진단 리포트 생성
