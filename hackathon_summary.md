# 해커톤 작업 요약 — [10] 마케팅 공통: 콘텐츠 성과 예측기

> 발표 자료 제작용 종합 요약본. 상세 근거는 `decisions.md`(의사결정 로그)·`explanation.md`(말로 풀어쓴 설명)·`output/content_prediction_report.md`(최종 리포트)·`output/strategy_plan.md`(전략 기획안)에 있음.

---

## 1. 과제 개요

- **트랙**: 마케팅(직무 공통) · 콘텐츠 마케터/그로스 마케터 채용 과제 · 개인 작업 3시간
- **문제**: 마케터는 콘텐츠 발행 전 "반응이 어떨까"를 감(感)으로 판단한다. 과거 성과 데이터가 있어도 매번 수동으로 유사 콘텐츠를 찾아 비교하기 번거롭고, A/B 테스트는 발행 후 비교라 발행 전 검증 수단이 없다.
- **미션**: 과거 콘텐츠 성과 데이터(70건, 정제 후 69건)를 패턴으로 학습해, 신규 콘텐츠 5건에 대해 **예상 CTR 범위 · 유사 콘텐츠 TOP3 비교 · 개선 제안 2가지**를 진단하는 `content_prediction_report.md`를 생성.
- **중요 제약**: ML 예측 모델이 아님(70행은 과적합 위험) → **유사 콘텐츠의 실제 성과로 기대 범위를 제시하는 패턴 기반 진단**으로 프레이밍.
- **달성 레벨**: 기본(필수: 진단 리포트) → 권장(재현 가능한 파이프라인) → **도전(발행 전략 기획안)까지 3단계 요구사항 모두 완료**.

## 2. 접근 방법 (파이프라인 구조)

Python(계산)과 Claude(해석·제안)의 역할을 분담:

1. **데이터 정제**: 완전 중복 C003 제거(70→69건), engagement_rate·reach 결측 2건은 컬럼 평균으로 대체, has_emoji 문자열→불리언 변환
2. **집계**: 채널·유형·시간대·이모지·주제별 CTR/인게이지먼트 평균을 `compute_aggregates()`로 실시간 계산
3. **유사도 계산**: type·channel 일치를 하드필터로, topic_category(+4)·has_emoji(+2)·posting_hour(+1) 가중 점수로 TOP3 선정
4. **개선 제안 자동 선정**: 발행시간·이모지·제목길이(블로그) 3개 레버의 CTR 개선폭(delta)을 전부 계산해 상위 2개 자동 채택 — 사람이 문구를 손으로 고르지 않음
5. **리포트 생성**: `render_report()`가 Markdown 리포트를 자동 조립 → `output/content_prediction_report.md`

전체 로직은 `predict_pipeline.py` 하나로 구현되어 있고, `python predict_pipeline.py [new_csv] [past_csv] [output]` 형태로 **새 CSV를 넣어도 동일 품질로 재현**된다(권장 요구사항). `.claude/skills/성과예측/SKILL.md`로 `/성과예측` 스킬 패키징까지 완료.

## 3. 핵심 설계 결정 & 근거 (11가지)

발표에서 "왜 이렇게 만들었나요?" 질문에 바로 답할 수 있도록 정리(전문은 `explanation.md`):

| # | 결정 | 핵심 근거 |
|---|---|---|
| 1 | type·channel = 하드필터 | 형식이 다르면 CTR 분포 자체가 달라 비교가 무의미 (숏폼 vs 블로그 비교 불가) |
| 2 | topic(4):emoji(2):hour(1) 가중치 | 과제 힌트의 "완화 순서(hour→emoji→topic)"를 거꾸로 뒤집어 중요도로 환산, 2배씩 차등을 둬 상위 조건 1개 일치가 하위 조건 전부 일치보다 항상 우선하도록 수학적으로 보장 |
| 3 | 동점 시 headline_length 차이로 타이브레이크 | 점수가 같을 때 그나마 제목 길이라도 비슷한 쪽을 우선 |
| 4 | 결측 2건(C069 engagement_rate, C070 reach) → 컬럼 평균 대체 | 표본 손실을 피하면서 왜곡을 최소화 |
| 5 | 완전 중복 C003 제거 | 이중 반영으로 인한 통계 왜곡 방지 |
| 6 | 개선 제안: delta 큰 순 자동 상위 2개 선정 | "감이 아닌 데이터 근거" 원칙 준수 + 새 CSV에도 동일 로직 재현 가능 |
| 7 | 리포트 섹션 순서: TOP3표 → CTR범위 → 개선제안 | 회의·보고 상황에서 "결론 먼저"보다 "증거 먼저"가 설득력 있음 (판사가 증거 없이 판결부터 내리면 안 되는 것과 같은 논리) |
| 8 | 신규1(이미 최고 조합)은 "유지 권장"을 1번 제안으로, 세부 레버를 2번으로 보충 | 원래 로직은 "더 나은 조합 찾기"라 1등 조합엔 추천할 게 없어 제안이 1개만 나오는 결함을 폴백으로 해결 |
| 9 | 저신뢰 기준: "2점 이상 후보 0건" → "3건 미만"으로 조정 | 신규5는 2점 이상 후보가 1건 있어 "0건" 기준을 통과했지만, TOP3를 채울 만큼(3건) 좋은 후보가 없는 것 자체가 문제였음. 실제 검증(신규1~4는 85~100%가 2점 이상, 신규5만 12.5%) 후 기준 재정의 |
| 10 | 리포트=결과 중심, decisions.md=과정·근거 중심으로 역할 분담 | 리포트에 설계 과정을 다 넣으면 마케터가 봐야 할 핵심(CTR·제안)이 묻힘 |
| 11 | 개선 제안 산출을 "채널" 단위로 집계(유형×채널 아님) | 유형×채널까지 쪼개면 일부 조합 표본이 1~2건으로 줄어 평균 신뢰도 저하 |

## 4. 진단 결과 요약 (신규 콘텐츠 5건)

| 신규 콘텐츠 | 조건 | 예상 CTR | 핵심 판단 |
|---|---|---|---|
| 1. 주니어 마케터 6개월 성장기 | 숏폼·유튜브·20시 | 3.5~5.2% | 숏폼×유튜브가 전체 1위 조합(4.38%) → 유지 권장 |
| 2. 노션으로 끝내는 일주일 업무 정리 | 카드뉴스·인스타그램·18시 | 3.4~4.0% | 20시로 조정 시 +0.48%p 기대 |
| 3. 2026 디지털 마케팅 트렌드 가이드 | 블로그·8시·34자 | 1.8~4.0% | 제목 30자 미만 축약 시 +1.74%p 기대 (5건 중 개선 여지 최대) |
| 4. 수강생 취업 성과 한눈에 보기 | 인포그래픽·인스타그램·18시 | 3.2~3.4% | 20시 조정 시 +0.48%p 기대 |
| 5. 점심에 보는 AI 트렌드 요약 | 숏폼·인스타그램·12시 | 3.2~4.6% (저신뢰) | 벤치마크 후보 8건 중 진짜 유사 1건뿐 → 20시+이모지 조정 후 발행 권장 |

**종합 발행 우선순위**
- ✅ **즉시 발행**: 신규1, 신규2 (신뢰도 높음, CTR 하단 3.4% 이상)
- 🔧 **개선 후 발행**: 신규3(제목 축약), 신규4(시간 조정)
- ⚠️ **조건부 발행**: 신규5 (저신뢰 — 시간·이모지 조정 후 발행, 발행 후 실측치로 검증 필요)

**공통 인사이트**: 저녁(18~20시) 발행 + 제목 이모지는 채널 불문 CTR이 대체로 높음. 블로그는 제목 30자 초과 시 CTR 급락.

## 5. Challenge: 콘텐츠 발행 전략 기획안 (`output/strategy_plan.md`)

**우선순위 공식(본인 설계)**: CTR 임팩트(45%) + 적용 범위(35%) + 실행 난이도(20%), 100점 만점, 70점 이상 P0

| 기획안 | 내용 | CTR 임팩트 | 적용 범위 | 실행 난이도 | 총점 | 우선순위 |
|---|---|---|---|---|---|---|
| ① 소셜 저녁 발행(18~20시) + 이모지 표준화 | 실측 결합효과 +0.99%p | 26/45 | 25/35 (소셜 50/69건, 72.5%) | 20/20 | **71** | **P0** (동점 타이브레이크: 난이도 높음) |
| ② 블로그 제목 30자 룰 | 실측 -1.74%p | 45/45 | 10/35 (블로그 19/69건, 27.5%) | 16/20 | **71** | P1 |

동점(71점)이라 실행 난이도가 더 쉬운 ①을 P0로 유지하되, **두 기획안을 동시 진행**하는 것을 최종 권장.

> ⚠️ **주의 (발표 전 확인 필요)**: `explanation.md`의 9번 항목 표는 재검증 이전 버전 수치(①91점/②56점)를 그대로 담고 있어 위 최신 수치(71/71 동점)와 다릅니다. 발표 자료에는 `strategy_plan.md` 최종본 기준인 **71점 동점**을 사용하세요.

## 6. 검증 과정에서 발견·수정한 계산 오류 (발표 시 신뢰도 강조 포인트)

작업 중 스스로 재검증하며 발견해 정정한 사례들 — "숫자를 검증 없이 쓰지 않았다"는 근거로 발표에 활용 가능:

1. **이론상 상한 vs 실측 결합효과 혼동**: 발행시간(+1.48%p)과 이모지(+1.06%p) 효과를 단순 합산해 +2.54%p로 제시했다가, 두 조건이 독립이 아님(소셜 콘텐츠의 74%가 이미 두 조건 동시 충족)을 확인하고 실측 결합효과 **+0.99%p**로 정정.
2. **적용 범위 기준 혼용(type vs channel)**: 기획안1 적용범위를 `type≠블로그` 기준(52건, 75.4%)으로 잘못 계산했다가, 인포그래픽이 channel=블로그/유튜브로 발행되는 케이스가 있어 실제 `channel` 기준(50건, 72.5%)으로 정정. 기획안2도 동일하게 17건→19건으로 정정.
3. **신규5 TOP3 저점수 원인 오진단 방지**: "정렬 로직 버그 아닌가" 의심을 실제 8개 후보 재계산으로 검증 → 버그가 아니라 해당 조건 풀 자체에 진짜 유사 콘텐츠가 부족한 구조적 상황임을 확인, 저신뢰 판정 기준을 데이터로 재정의.
4. **개선 제안 근거 표본 반전 발견**: 신규1의 발행시간 제안(n=3)이 전체 69건 기준으로는 방향이 반대(20시>18시)임을 발견 → 삭제하지 않고 "표본이 적어 참고용" + "전체 기준 반전" 캐벗을 자동으로 붙이는 로직 추가.

## 8. 발표 구성 제안

1. 문제 정의: 마케터의 "감으로 판단" 문제 (1번 항목)
2. 왜 ML이 아닌 패턴 기반 진단인가 (제약 조건)
3. 파이프라인 데모: 정제 → 유사도 → 제안 → 리포트 (2번 항목)
4. 핵심 설계 결정 3~4개 선별 소개 — 하드필터, 가중치 4:2:1, 저신뢰 기준 (3번 항목)
5. 결과: 신규 5건 진단 + 발행 우선순위 (4번 항목)
6. Challenge: 전략 기획안과 우선순위 공식 (5번 항목)
7. **차별화 포인트**: 검증 과정에서 스스로 발견·수정한 계산 오류 사례 (6번 항목) — 데이터 근거 기반 사고를 보여주는 가장 강력한 소재
